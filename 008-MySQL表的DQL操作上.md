## MySQL表的DQL操作上--结果排序

查询过后可以根据某些规则排序：

排序规则如下：

```sql
select 列名1,列名2,列名n 
from 表名
where 条件
order by 列名1 [asc/desc],列名1 [asc/desc]...;
```

1. 单列排序

```sql
示例：
select * from product order by price asc; => 按价格升序
结果如下：
+----+-----------+-------+
| id | name      | price |
+----+-----------+-------+
|  1 | 罗技m1    |    20 |
|  2 | 罗技m2    |    20 |
|  3 | 罗技m3    |    30 |
|  4 | 罗技m4    |    30 |
|  5 | 罗技m5    |    30 |
|  6 | 罗技m6    |    40 |
|  7 | 罗技m7    |    40 |
|  8 | 罗技m8    |    50 |
|  9 | 罗技m9    |    50 |
| 10 | 罗技m10   |    50 |
| 11 | 罗技m11   |    50 |
| 12 | 罗技m12   |    60 |
| 13 | 罗技m13   |    60 |
+----+-----------+-------+
select * from product order by price desc; => 按价格降序
结果如下：
+----+-----------+-------+
| id | name      | price |
+----+-----------+-------+
| 12 | 罗技m12   |    60 |
| 13 | 罗技m13   |    60 |
|  8 | 罗技m8    |    50 |
|  9 | 罗技m9    |    50 |
| 10 | 罗技m10   |    50 |
| 11 | 罗技m11   |    50 |
|  6 | 罗技m6    |    40 |
|  7 | 罗技m7    |    40 |
|  3 | 罗技m3    |    30 |
|  4 | 罗技m4    |    30 |
|  5 | 罗技m5    |    30 |
|  1 | 罗技m1    |    20 |
|  2 | 罗技m2    |    20 |
+----+-----------+-------+
```

2. 多列排序

```sql
在前面的规则下再根据后面的规则排序
示例：
select * from product order by price desc,id desc; => 先按价格降序排，若价格一样就按id降序排
+----+-----------+-------+
| id | name      | price |
+----+-----------+-------+
| 13 | 罗技m13   |    60 |
| 12 | 罗技m12   |    60 |
| 11 | 罗技m11   |    50 |
| 10 | 罗技m10   |    50 |
|  9 | 罗技m9    |    50 |
|  8 | 罗技m8    |    50 |
|  7 | 罗技m7    |    40 |
|  6 | 罗技m6    |    40 |
|  5 | 罗技m5    |    30 |
|  4 | 罗技m4    |    30 |
|  3 | 罗技m3    |    30 |
|  2 | 罗技m2    |    20 |
|  1 | 罗技m1    |    20 |
+----+-----------+-------+
```

3. 按列别名排序

```sql
示例：
select price money from product order by money desc;
结果如下+-------+
| money |
+-------+
|    60 |
|    60 |
|    50 |
|    50 |
|    50 |
|    50 |
|    40 |
|    40 |
|    30 |
|    30 |
|    30 |
|    20 |
|    20 |
+-------+
注：
别名如果使用''包裹排序无效。
```

4. DQL语句执行顺序

```sql
1. 先from
2. 再where
3. 再select
4. 最后order by
```
