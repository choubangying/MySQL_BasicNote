## MySQL的DQL操作上--单表过滤查询

过滤查询的格式：

```sql
select 列名1，列名2，列名n
from 表名
where 条件;
```

1. 比较运算符

```sql
>              大于
<              小于
=              等于
>=             大于等于
<=             小于等于
!=(<>)         不等于

示例：
select * from product where price = 20;
查询结果：
+----+----------+-------+
| id | name     | price |
+----+----------+-------+
|  1 | 罗技m1   |    20  |
|  2 | 罗技m2   |    20  |
|  8 | 罗技m8   |    20  |
+----+----------+-------+
若条件是字符串需''包裹：
select * from product where name = '罗技m1';
条件可以是表达式：
select * from product where price * 0.5 > 10;

注：
执行顺序：
1.先from
2.再where
3.最后select
```

2. 逻辑运算符

```
and(&&)
or(||)
not(!)
示例：
select * from product where price <= 50 and price >= 40;
结果如下：
+----+-----------+-------+
| id | name      | price |
+----+-----------+-------+
|  6 | 罗技m6    |    40 |
|  7 | 罗技m7    |    40 |
|  9 | 罗技m9    |    50 |
| 10 | 罗技m10   |    50 |
| 11 | 罗技m11   |    50 |
+----+-----------+-------+
```

3. 优先级规则

```
优先级       运算符
1           所有的比较运算算符
2           not
3           and
4           or

注;有括号先算括号
```

4. 范围查询

```sql
示例：
select * from product where price between 20 and 40;
结果如下：
+----+----------+-------+
| id | name     | price |
+----+----------+-------+
|  1 | 罗技m1   |    20 |
|  2 | 罗技m2   |    20 |
|  3 | 罗技m3   |    30 |
|  4 | 罗技m4   |    30 |
|  5 | 罗技m5   |    30 |
|  6 | 罗技m6   |    40 |
|  7 | 罗技m7   |    40 |
+----+----------+-------+
```

5. 集合查询

```sql
示例：
select * from product where price in(20,40);
结果如下：
+----+----------+-------+
| id | name     | price |
+----+----------+-------+
|  1 | 罗技m1   |    20 |
|  2 | 罗技m2   |    20 |
|  6 | 罗技m6   |    40 |
|  7 | 罗技m7   |    40 |
+----+----------+-------+
注：
集合查询是单个匹配集合中的元素，并不是匹配区间
```

6. 空值查询

```sql
先向product表中新增一个数据项，然后查询
insert into product (name) values ('jaychou');
示例：
select * from product where price is null;
结果如下：
+----+---------+-------+
| id | name    | price |
+----+---------+-------+
| 14 | jaychou |  NULL |
+----+---------+-------+
```

7. 模糊查询

```
模糊查询用于在知道一些关键词的前提下来做查询,有两种方式‘
1. _方式，表示模糊匹配1个字符
2. %方式，表示模糊0个到n个字符

示例：
select * from product where name like '__m_'; =>查询name为xxmx格式的货物
结果如下：
+----+----------+-------+
| id | name     | price |
+----+----------+-------+
|  1 | 罗技m1   |    20 |
|  2 | 罗技m2   |    20 |
|  3 | 罗技m3   |    30 |
|  4 | 罗技m4   |    30 |
|  5 | 罗技m5   |    30 |
|  6 | 罗技m6   |    40 |
|  7 | 罗技m7   |    40 |
|  8 | 罗技m8   |    50 |
|  9 | 罗技m9   |    50 |
+----+----------+-------+
select * from product where name like '%m%';
结果如下：
+----+-----------+-------+
| id | name      | price |
+----+-----------+-------+
|  1 | 罗技m1    |    20 |
|  2 | 罗技m2    |    20 |
|  3 | 罗技m3    |    30 |
|  4 | 罗技m4    |    30 |
|  5 | 罗技m5    |    30 |
|  6 | 罗技m6    |    40 |
|  7 | 罗技m7    |    40 |
|  8 | 罗技m8    |    50 |
|  9 | 罗技m9    |    50 |
| 10 | 罗技m10   |    50 |
| 11 | 罗技m11   |    50 |
| 12 | 罗技m12   |    60 |
| 13 | 罗技m13   |    60 |
+----+-----------+-------+
```
